## 3.2. User Stories
|**Epica ID**|**Descripción**|
| :- | :- |
|EP001|Como productor agrícola, quiero que la aplicación haga un monitoreo de la humedad de mi parcela, para saber cuando tengo que regar mis forrajes.|
|EP002|Como usuario, quiero recibir un reporte detallado del estado actual de la parcela para tener un mejor control sobre el uso de agua|
|EP003|Como productor agricola, quiero tener un control total sobre las modalidades de riego, para que se adapte a mi tiempo|
|EP004|Como productor agrícola, quiero tener opciones de suscripción para ajustarla a mis necesidades.|
|EP005|Como productor agricola quiero una aplicación móvil y web que me ayude a controlar mis forrajes|

|Story ID|Título|Descripción|Criterios de Aceptación|Relacionado con (Epic ID)|
| - | - | - | - | - |
|US001|Monitoreo de Humedad en Parcela|Como productor agrícola, quiero que la aplicación mida la humedad de mi parcela en tiempo real, para determinar el momento adecuado para el riego.|<p>-**Dado** que la aplicación está configurada, **cuando** la humedad de la parcela está por debajo del umbral, **entonces** la aplicación notifica al usuario.</p><p>- **Dado** un sensor de humedad instalado, **cuando** la medición falla, **entonces** se genera una alerta para el usuario.</p>|EP001|
|US002|Generación de Reporte de Humedad	|Como usuario, quiero que la aplicación genere un reporte detallado del estado actual de la humedad de la parcela, para un mejor control del riego.	|<p>- **Dado** que el usuario solicita un reporte, **cuando** se procesa la solicitud, **entonces** se muestra un reporte detallado de la humedad de la parcela.</p><p>- **Dado** que el usuario recibe el reporte, **cuando** el nivel de humedad es crítico, **entonces** el reporte debe incluir recomendaciones de acción.</p>|EP002|
|US003|Configuración de Métodos de Riego	|Como productor agrícola, quiero configurar el tipo de riego según mi disponibilidad de tiempo, para optimizar el uso del agua y el tiempo de riego.	|<p>**Dado** que el usuario selecciona un método de riego, **cuando** el método es seleccionado, **entonces** la aplicación ajusta el riego según el método.</p><p>- **Dado** que el riego se está ejecutando, **cuando** el tiempo de riego es alcanzado, **entonces** el riego se detiene automáticamente</p>|EP003|
|US004|Selección de Modalidad de Riego Manual o Automático	|Como productor agrícola, quiero alternar entre riego manual o automático, para ajustar el riego a las necesidades específicas de mis cultivos.	|<p>- **Dado** que el usuario tiene habilitada la opción de riego, **cuando** selecciona "Manual", **entonces** la aplicación permite iniciar y detener el riego manualmente.</p><p>- **Dado** que el usuario selecciona "Automático", **cuando** las condiciones de riego son óptimas, **entonces** el riego se inicia automáticamente</p>|EP003|
|US005|Suscripción a Planes Personalizados	|Como productor agrícola, quiero suscribirme a diferentes planes, para elegir el que mejor se adapte a mis necesidades y presupuesto.	|<p>- **Dado** que el usuario está en la página de suscripción, **cuando** selecciona un plan, **entonces** el sistema debe guardar la suscripción del usuario.</p><p>- **Dado** que el usuario tiene una suscripción activa, **cuando** selecciona cambiar de plan, **entonces** la aplicación permite realizar el cambio sin perder los datos actuales.</p>|EP004|
|US006|Notificación de Estado de Parcela	|Como usuario, quiero recibir notificaciones sobre el estado de mi parcela, para estar siempre informado y tomar decisiones oportunas.	|<p>- **Dado** que el usuario tiene habilitadas las notificaciones, **cuando** el estado de la parcela cambia, **entonces** el usuario recibe una notificación.</p><p>- **Dado** que la notificación es recibida, **cuando** el usuario la abre, **entonces** se muestran los detalles de los cambios en el estado de la parcela.</p>|EP002|
|US007|Landing Page Informativa	|Como visitante del sitio web, quiero ver una página de aterrizaje que explique los beneficios de la solución Thirsty Seed, para entender mejor el producto.	|<p>-**Dado** un visitante accede a la Landing Page, **cuando** carga la página, **entonces** se muestra información clara y concisa sobre los beneficios del producto.</p><p>- **Dado** que el visitante quiere más información, **cuando** hace clic en "Aprender Más", **entonces** se redirige a una sección con detalles adicionales.</p>|EP004|
|US008|Integración con Sensores de Humedad	|Como developer, quiero integrar sensores de humedad con la aplicación, para asegurar que los datos de humedad sean precisos y en tiempo real.	|<p>- **Dado** que un sensor está conectado, **cuando** se realiza una solicitud de datos, **entonces** el sistema debe recibir y procesar los datos en menos de 5 segundos.</p><p>- **Dado** que el sensor falla en responder, **cuando** se intenta de nuevo, **entonces** el sistema debe manejar el error y reintentar la solicitud hasta tres veces.</p>|EP001|
|US009|API RESTful para Reportes	|Como developer, quiero desarrollar una API RESTful para el manejo de reportes, para que los usuarios puedan acceder a los datos desde diferentes plataformas.	|<p>- **Dado** un endpoint para reportes está disponible, **cuando** un cliente envía una solicitud GET, **entonces** el sistema devuelve los datos de los reportes en formato JSON.</p><p>- **Dado** una solicitud POST se envía, **cuando** los datos son válidos, **entonces** el sistema guarda el reporte en la base de datos.</p>|EP002|
|US010|Personalización de Notificaciones|Como usuario, quiero personalizar las notificaciones que recibo sobre el estado de la parcela, para enfocarme en la información más relevante para mi operación.|<p>**- Dado** que el usuario accede a la configuración de notificaciones, **cuando** selecciona los tipos de notificación, **entonces** solo recibe las seleccionadas.</p><p></p><p>- **Dado** que el usuario accede a la configuración de notificaciones, **cuando** selecciona un tipo de notificación pero la aplicación falla en guardar la selección, **entonces** el usuario no debería recibir notificaciones no deseadas.</p>|EP002|
|US011|Historial de Riego|Como productor agrícola, quiero acceder a un histórico detallado de las actividades de riego, para analizar patrones y mejorar la planificación futura.|<p>- **Dado** que el usuario accede a la sección de historial, **cuando** selecciona un periodo de tiempo, **entonces** se muestra un reporte detallado de todas las actividades de riego en ese periodo.</p><p></p><p>-  **Dado** que el usuario accede a la sección de historial, **cuando** selecciona un periodo de tiempo y no hay datos disponibles, **entonces** la aplicación debe notificar que no se encontraron registros y sugerir verificar otro periodo.</p>|EP002|
